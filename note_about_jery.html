Approach 1 — Letta Native Messaging

Use native Letta messaging for simple requests when you want the easiest path.

  agent_id="agent-1caa2fff-5188-4f7e-996b-34146de966e3",
  message={
    "task": "analyze_statement",
    "context": {
      "statementPath": "/home/adamsl/rol_finances/readable_documents/bank_statements/january/sample.pdf"
    }
  }
)

Approach 2 — A2A JSON-RPC

Use A2A for structured workflows, task polling, and explicit lifecycle control.

POST http://10.0.0.143:9001/

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "SendMessage",
  "params": {
    "skill": "analyze_statement",
    "input": {
      "statementPath": "/home/adamsl/rol_finances/readable_documents/bank_statements/january/sample.pdf"
    }
  }
}
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "GetTask",
  "params": {
    "taskId": "<task-id>"
  }
}

Lifecycle: SUBMITTED -> WORKING -> COMPLETED or FAILED.

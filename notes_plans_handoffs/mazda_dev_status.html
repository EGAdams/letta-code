3a. Where the Self-Improvement Loop Fits Now

The self-improvement system built in Milestones 0–7 + Hardening Phases 8–9 was always designed finance-blind: AgentKernel never imports finance code or Mazda directly, and reaches "the agent being improved" only through IAgentPackageFactory / IAgentRunner ports. That boundary is exactly what lets this loop carry over cleanly to the new orchestration design — only the thing being wrapped changes.

Self-improvement conceptOld usage (discarded)New usage (this design)
IAgentRunner / AgentKernel.run_task Run Mazda against a finance statement, using the 7 finance tools Run Mazda's orchestration step: delegate to one or more minions and collect their Claude Agent SDK session results
TraceRepository / evidence store Trace finance tool calls (verify_statement_totals, check_duplicates, ...) Trace Mazda→minion delegation calls and each minion's Claude Agent SDK session (inputs, outputs, tool calls, token usage)
IEvaluator / FinanceVerdictJudge Finance-rule verdicts (totals match, vendor known, category correct, ...) Generalize to an orchestration verdict: did the minion's SDK session complete the delegated task, with what quality/cost — finance-specific verifiers become one pluggable IEvaluator family among others, not the only one
WrapperRevision / prompt & toolset versioning Versioned prompts/tools for Mazda's finance-tool wrapper Versioned prompts/configs for Mazda's delegation logic and for each minion's Claude Agent SDK task templates (system prompts, allowed tools, SDK invocation parameters)
ProposalGenerator / ExperimentRunner / GateChain Propose & gate changes to Mazda's finance prompts/tools Propose & gate changes to Mazda's minion-delegation strategy or a minion's SDK task template, A/B'd via baseline-vs-candidate experiments before activation
ActivationService / RollbackService / snapshots Activate/rollback finance-tool wrapper revisions Activate/rollback Mazda's or a minion's orchestration-prompt/SDK-template revisions — human-approved, rollback-safe, unchanged mechanics
Net effect: none of the 145 unit tests or the Milestone 0–7 / Phase 8–9 implementation needs to be thrown away. The agent_self_improvement package (contracts + implementations) stays as the control plane. What gets rewritten is the agent package / runner adapter layer that previously targeted "Mazda + 7 finance tools" and now targets "Mazda + minions driving the Claude Agent SDK." The integration-certification backlog (8 test-code fixes → P0 suite, from the prior status) still applies to the core loop and is still the right next step regardless of which agent package sits on top of it.

1. What Mazda Is (current direction)

  • Mazda is the orchestrator — no "Orchestrator" suffix, just Mazda (agent-6b536cf4-ec88-4290-b595-fed21d14bd8e).
  • Mazda delegates to 5 minions (Letta agents) via send_letta_message. Each minion's specialty is driving a Claude Agent SDK (TypeScript) session for its slice of work.
  • Finance domain logic (totals/vendor/category/duplicate verifiers) is exposed to those SDK sessions as MCP tools — not called by Mazda directly.
Discarded (do not implement): the older "give Mazda 7 direct finance tools" design and the team_construction_plan.html "Mazda Orchestrator" (IAgentTeamFactory/StageEnvelope/deterministic service family). Superseded by the direction above.
Carried forward (not discarded): the self-improvement loop (AgentKernel, trace/evidence store, verdicts, proposals, baseline-vs-candidate experiments, gates, activation/rollback, HTML reports — Milestones 0–9, 145 unit tests). It is finance-blind, generic infrastructure and stays as the control plane. Only the agent-package / runner-adapter layer on top changes — see §4.

4. Self-Improvement Loop — how it fits now

The Milestone 0–9 loop never imports finance code or Mazda directly — it reaches "the agent being improved" only through IAgentPackageFactory / IAgentRunner ports. That boundary lets it carry over unchanged; only the thing being wrapped changes from "Mazda + 7 finance tools" to "Mazda + minions driving the Claude Agent SDK."

Loop conceptNew usage (this design)
IAgentRunner / AgentKernel.run_task Run Mazda's orchestration step: delegate to minions, collect their SDK session results
TraceRepository / evidence store Trace Mazda→minion delegations + each minion's SDK session (inputs, outputs, tool calls, tokens)
IEvaluator family Orchestration verdict (did the session complete, at what quality/cost); finance verifiers become one pluggable evaluator
WrapperRevision Versioned Mazda delegation logic + per-minion SDK task templates (prompts, allowed tools, params)
Proposal/Experiment/Gate + Activation/Rollback Propose, A/B, gate, and human-approve changes to delegation strategy or a minion's SDK template; rollback-safe

None of the 145 unit tests are thrown away. The integration backlog ("8 test-code fixes → P0 suite") still applies to the core loop and is a lower-priority next step (§5).

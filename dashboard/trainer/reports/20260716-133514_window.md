That raw text actually contained `bp_site_id_2836900`, an exact match for the known vendor key `bp_2836900_leo_gas` → category 374 ("Leo Gas") in `vendor_category.yaml`. Expense 1504 was stored with the wrong category.

This is a **wrapper defect**, not a model competence issue: the instructions don't tell Mazda how to construct a clean `vendor_key` input when `merchant_name` is present but the raw text is dominated by receipt boilerplate.

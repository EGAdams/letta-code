Add Early Warning Tests and Dashboard Status Alerts

We need a way to catch this type of problem earlier and warn the team directly from the dashboard.

Do not apply the production fix yet. Create the failing tests first.

Goal

Add unit tests and integration tests that detect this problem before it reaches users. The tests should fail now, before the implementation fix is applied.

Unit Test Requirements

Create unit tests that catch this type of error in the future.

  • Reproduce the current failure.
  • Fail before the fix is applied.
  • Verify the expected behavior.
  • Cover important edge cases.
  • Verify that the system correctly detects the problem condition.
  • Verify that healthy systems are not incorrectly marked as broken.

After writing the unit tests, run them and confirm they fail for the correct reason.

Integration Test Requirements

After the unit tests are written, create integration tests that verify the dashboard warning behavior.

  • When this type of problem occurs, the main Status tab/button should blink yellow and white.
  • When the user clicks the Status tab/button, the related status section should also blink yellow and white.
  • Healthy status sections should be green.
  • Remove the current blue color from these Status tabs.
  • Warning-level problems should use yellow and white blinking.
  • Critical problems should use red and white blinking.

Status Color Rules

  • Healthy: Green
  • Warning: Blinking yellow and white
  • Critical: Blinking red and white

Example: if the Server status is critically unhealthy, the Server status section should blink red and white.

If the system detects an early warning but not a critical failure, the affected status section should blink yellow and white.

Expected Workflow

  1. Write the unit tests first.
  2. Run the unit tests and confirm they fail.
  3. Write the integration tests.
  4. Run the integration tests and confirm they fail.
  5. Do not apply the actual production fix yet.
  6. After we review the failing tests, we will decide on the implementation fix.
